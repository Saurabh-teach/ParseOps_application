1. Introduction

What is ParseOps?

ParseOps is a modern project
management and team operations platform designed
for organizations that need structured planning, fair task distribution, and
clear execution visibility.

 

Inspired by products such as
ClickUp, ParseOps combines workspace management,
goals, tasks, intelligent scheduling, time tracking, reusable templates,
contextual chat, notifications, and analytics in a single multi-page
experience.

 

Every workspace is isolated from
other workspaces. Users may belong to multiple
organizations, with a different role and permission set in each one.

 

Key features

● Multi-organization workspace support
with Owner, Admin, and Member roles

● Row-level organization isolation for
workspace data

● Custom permission overrides for
individual workspace members

● Goals, key results, linked tasks,
progress tracking, and dependencies

● Intelligent task scheduling with
priority scoring and Smart Suggest

● Strict one-active-task-per-member
rule within each workspace

● Pending Queue for unassigned work

● Recently Freed Members
recommendations using a 15-minute completion window

● Time tracking through task tickets
with start, pause, resume, and accumulated time

● Task creation using impact, risk,
priority, urgency, and effort

● Workload-aware due-date planning
with leave and overload warnings

● CSV and Excel template import for
goals, folders, documents, and tasks

● Contextual real-time chat for goals
and tasks

● Direct and group chat, reactions,
replies, typing indicators, and read status

● Workspace calendar, notes,
notifications, analytics, leave management, and recovery

Multi-tenancy model

ParseOps uses row-level
multi-tenancy. A shared database stores all
organizations, while organization-scoped records include an organization
foreign key.

 

Access is controlled in two
stages:

 

1. The system confirms the authenticated user has an active
OrganizationMembership
for the requested workspace.

2. Role, custom permission, ownership, assignment, and
sharing rules determine
whether the user may view or change the requested record.

 


 


 



Important:
Membership and permission checks must always include the
organization context. A user who can access one workspace must never gain
access to another workspace's records.

Organization and membership

An Organization is the top-level tenant
boundary. A user accesses a workspace
through one unique OrganizationMembership.

 

Important organization fields:

 

● id: UUID primary key

● name and slug: workspace identity

● owner: primary owning user

● is_active: supports deactivation and reactivation

● is_public: controls discoverability

● onboarding_completed: records workspace setup
progress

 

Important membership fields:

 

● organization and user

● role: owner, admin, or member

● custom_permissions: per-member permission overrides

● is_active: controls current workspace access

● invited_by and joined_at

 

The system prevents deletion or
demotion of the last active workspace owner.

Custom permissions

Owners and admins can apply
custom permission overrides to members. Examples
include:

 

● Create workspace goals or tasks

● View all goals or tasks

● Edit or delete goals and tasks

● Manage goal or task visibility

● Assign goals or tasks

● Manage comments and attachments

● Manage workspace settings

 

Owner authority is protected.
Admins cannot modify an owner's permissions or
manage peer admins where owner authority is required.

5. Core Features Documentation

5.1 Organization and workspace management

A workspace groups members,
goals, tasks, templates, chats, notifications, and
analytics under one tenant boundary.

 

Owners and admins can:

 

● Configure workspace information

● Invite members with a selected role

● Review join requests

● Remove or restore members

● Review workspace history

● Manage member permissions

 

Owners can additionally:

 

● Promote or demote roles while
protecting the final active owner

● Deactivate and reactivate the
workspace

● Permanently delete the workspace

 

When a member is removed, their
membership is marked inactive. They can still
sign in globally but immediately lose access to that workspace, including its
WebSocket chat connection.

 

5.2 Goals management

Goals represent strategic
outcomes. Each goal can include:

 

● An owner and multiple assignees

● Start and due dates

● Priority, timeframe, and status

● Parent goals and dependencies

● Key results

● Linked tasks

● Nested folders and
template-generated content

● Organization, specific-user, or
private sharing

● A dedicated contextual chat room

 

Goal progress behavior:

 

1. If the goal has key results, progress is the average
key-result progress.

2. If no key results exist, progress is based on linked task
completion.

3. At 100%, the goal becomes completed.

4. Between 0% and 100%, the goal becomes in_progress.

 

5.3 Task management

Tasks are executable units of
work. They may be created independently or linked
to a goal.

 

Task creation

A task can include:

 

● Title and description

● Task, story, or bug type

● Goal link

● Manual assignee or Smart Suggest
selection

● Watchers and shared viewers

● Priority, impact, risk, and
estimated effort

● Start date, due date, due time, and
reminders

● Sharing and visibility settings

 

If a task is created without an
assignee, ParseOps attempts to run the
scheduling engine for pending unassigned tasks in that workspace.

 

Assignment

The current task model uses one
responsible assignee. Assignment is validated
against active workspace membership.

 

Members cannot assign tasks to
owners or admins through the regular member
assignment flow. Assignment and reassignment actions remain subject to role,
custom permission, creator, and goal-owner checks.

 

Task lifecycle


 


 

Tasks also support comments,
threaded replies, mentions, attachments, extension
requests, feedback, submissions, reminders, overdue detection, soft deletion,
and recovery.

 

5.4 Intelligent Task Scheduling Engine

The scheduling engine
prioritizes unassigned work and ranks active workspace
members using task importance, employee suitability, fatigue, and workload
balance.

 

Scheduling inputs

Task scoring considers:

 

● Impact

● Priority

● Risk

● Urgency based on due date

● Estimated effort

 

         
    Task Score
=    
      (0.35 ×
Impact + 0.25 × Priority + 0.20 × Risk + 0.20 × Urgency)    
      ÷
sqrt(Effort)    
         

 

Employee scoring considers:

 

● Past performance

● Experience

● Efficiency

● Availability and approved leave

● Current workload

 

The final assignment score
applies fatigue and workload balancing:

 

         
    Load
Balance Factor = max(0.2, 1 - Assigned Hours / 8)    
         
    Final
Match Score =    
      Task
Score × Employee Score × (1 - Fatigue Penalty) × Load Balance Factor    
         

 

Scheduler flow


 


 

Smart Suggest

Smart Suggest is available while
creating or editing tasks. It:

 

1. Builds a temporary task profile using effort, priority,
impact, and risk.

2. Scores active members against that task.

3. Ranks candidates by match score.

4. Displays reasons such as low workload, excellent
performance, experience,
balanced score, or busy status.

5. Allows an authorized user to choose the recommendation or
manually select a
different member.

 

Smart Suggest is advisory. The
user making the assignment remains responsible
for reviewing availability and the reason shown.

 

One Member = One Active Task

ParseOps enforces a strict focus
rule within each workspace:

 



A member
cannot start a second task while another task or task ticket is
already in_progress in
the same workspace.



 

To start another task, the
member or an authorized manager must first pause the
current task, move it to another status, or complete it.

 

This rule applies when:

 

● Updating a task to in_progress

● Starting or resuming a task ticket

● Quick-assigning work where an
in-progress conflict exists

 

Fairness approach

The scheduler promotes fair
distribution by:

 

● Rejecting members who already have
an active in-progress task during automatic scheduling

● Excluding members at or above the
6.5-hour active-capacity threshold

● Reducing scores as workload and
fatigue increase

● Considering leave and availability

● Recalculating availability after
every assignment

● Showing busy status and explanatory
reasons in suggestions

 

5.5 Pending Queue and bulk scheduling

The Pending Queue displays tasks
that:

 

● Have no assignee

● Are not complete

● Are not soft-deleted

 

Pending work is ordered by
priority in the current UI. Each queue item shows
task details, goal context, estimated effort, due date, and a suggested member
when available.

 

Authorized users can:

 

● Assign the suggested member

● Select another member

● Review why a member was recommended

● Refresh the queue after assignments

 

Automatic scheduling processes
the complete set of pending unassigned tasks,
orders them by Task Score, and assigns them sequentially when eligible members
are available.

 

5.6 Check Free Members: 15-minute logic

The Recently Freed Members
tool identifies members whose task ticket changed
to done within the
last 15 minutes.

 

For each recently freed member,
ParseOps:

 

1. Finds pending unassigned tasks.

2. Calculates the member's match score for each task.

3. Sorts recommendations from strongest to weakest.

4. Displays up to five recommended tasks.

5. Allows an authorized user to assign a recommendation
immediately.

 

This creates a short operational
window for quickly moving available capacity
to the next suitable task.

 

5.7 Time tracking system

Each assigned task has a TaskTicket for
the responsible member. The ticket
stores execution status and accumulated time_spent_minutes.

 

Start, pause, and resume

● Start: Move the ticket to in_progress.
The one-active-task rule is checked.

● Pause: Move the ticket back
to todo and add
the elapsed session time.

● Resume: Move the ticket to in_progress
again. Previously accumulated time remains.

● Review/complete: Move the
ticket to the appropriate later status.

 

The frontend displays a live
seconds counter while a ticket is in progress.
Elapsed session minutes are added to the persisted ticket time when the status
changes.

 

Additional time controls:

 

● Idle in-progress tickets may be
auto-paused after eight hours.

● The auto-pause adds eight hours to
the ticket and notifies the member.

● ParseOps can notify the member and
task creator when tracked time exceeds the
assigned effort budget.

● Owners and admins can update any
ticket; members can update their own tickets.

7. Best Practices and Tips

For Owners and Admins

● Create one workspace per operational
boundary that needs separate membership
and data isolation.

● Keep at least two trusted owners
when possible to avoid ownership lockout.

● Use Admin for operational managers
and reserve Owner for workspace governance.

● Review custom permissions regularly
instead of granting broad access permanently.

● Require impact, risk, priority, and
effort before using Smart Suggest.

● Review the Pending Queue daily to
prevent unassigned work from becoming invisible.

● Use Check Free Members after major
completions or during active delivery windows.

● Respect Smart Suggest as a
recommendation; confirm context, skill fit, and team
commitments before assigning.

● Use templates for repeatable project
structures and validate imports in a small
workspace before large-scale use.

● Review extension requests, overdue
alerts, time overruns, and leave schedules
before changing assignments.

● Use specific or private sharing only
when required; overly restrictive sharing
can reduce team visibility.

 

For Team Members

● Keep only one task in progress and
pause it before switching context.

● Update status as work moves from
execution to review, testing, and completion.

● Use comments and contextual chat to
keep decisions attached to the relevant work.

● Mention teammates in comments only
when their attention is required.

● Track time consistently so workload
and scheduling recommendations remain useful.

● Submit task proof, files, or links
before completing work when required.

● Request an extension early and
include a clear reason.

● Keep profile, availability, and
leave information current.

● Review notifications and workspace
history regularly.

 

Data quality tips

● Use clear, action-oriented task
titles.

● Estimate effort in hours or minutes
before assignment.

● Use impact to represent business
value and risk to represent delivery uncertainty.

● Set realistic due dates and respond
to overload warnings.

● Link tasks to goals when the work
contributes to a strategic outcome.

● Keep imported goal titles and
assignee emails consistent.
